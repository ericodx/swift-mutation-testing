## Quick Start

### Generate a configuration file

```bash
# Auto-detects project type, scheme, destination, test targets, and testing framework
swift-mutation-testing init
```

This writes `.swift-mutation-testing.yml` at the project root. Review and adjust the generated file before running. For SPM packages, `scheme` and `destination` are not required and will be omitted.

### Run mutation testing

```bash
swift-mutation-testing
```

### Run an SPM package

```bash
# SPM packages are auto-detected — no scheme or destination needed
swift-mutation-testing /path/to/my-package
```

### Run on macOS without a simulator

```bash
swift-mutation-testing \
  --scheme MyScheme \
  --destination "platform=macOS"
```

### Run on iOS Simulator

```bash
swift-mutation-testing \
  --scheme MyApp \
  --destination "platform=iOS Simulator,name=iPhone 16"
```

## Configuration File

Generate a starter file with:

```bash
swift-mutation-testing init
```

### Full reference

This is the format generated by `init`. Comments show the default value for each optional field.

```yaml
# swift-mutation-testing configuration
# All settings are optional. CLI flags override file values.

scheme: MyApp
destination: platform=iOS Simulator,name=iPhone 16

# Limit test execution to a specific target (recommended when the project has UI tests)
# testTarget: MyAppTests

# Per-mutant test timeout in seconds (default: 120 Xcode, 30 SPM)
timeout: 60

# Number of parallel workers (default: max(1, CPU count - 1))
# concurrency: 4

# Disable result cache (re-runs all mutants on every execution)
# noCache: true

# Report output paths
# output: mutation-report.json
# htmlOutput: mutation-report.html
# sonarOutput: sonar-report.json

# Source file glob patterns to exclude from mutation
# exclude:
#   - "**/Generated/**"

# Mutation operators — set active: false to disable
mutators:
  - name: RelationalOperatorReplacement
    active: true
  - name: BooleanLiteralReplacement
    active: true
  - name: LogicalOperatorReplacement
    active: true
  - name: ArithmeticOperatorReplacement
    active: true
  - name: NegateConditional
    active: true
  - name: SwapTernary
    active: true
  - name: RemoveSideEffects
    active: true
```

### Field reference

| Field | Type | Default | Description |
|---|---|---|---|
| `scheme` | String | — | Xcode scheme to build and test (required for Xcode projects, omit for SPM) |
| `destination` | String | — | `xcodebuild` destination specifier (required for Xcode projects, omit for SPM) |
| `testTarget` | String | — | Limit test execution to this target (`-only-testing`) |
| `timeout` | Number | `120` (Xcode) / `30` (SPM) | Per-mutant test timeout in seconds |
| `concurrency` | Number | `max(1, CPUs − 1)` | Number of parallel test workers |
| `noCache` | Boolean | `false` | Disable result caching |
| `output` | String | — | Path for Stryker-compatible JSON report |
| `htmlOutput` | String | — | Path for interactive HTML report |
| `sonarOutput` | String | — | Path for SonarQube generic coverage report |
| `sourcesPath` | String | project root | Root directory for Swift source file discovery |
| `exclude` | List | `[]` | Glob patterns for source files to skip |
| `mutators` | Block | all active | Operator list with per-entry `active: true/false` (generated by `init`) |
| `operators` | List | all | Allowlist of operator identifiers — simpler alternative to `mutators` |

### Operator identifiers

| Identifier | What it mutates |
|---|---|
| `RelationalOperatorReplacement` | `>` `>=` `<` `<=` `==` `!=` |
| `BooleanLiteralReplacement` | `true` ↔ `false` |
| `LogicalOperatorReplacement` | `&&` ↔ `\|\|` |
| `ArithmeticOperatorReplacement` | `+` `-` `*` `/` `%` |
| `NegateConditional` | wraps conditions in `!()` |
| `SwapTernary` | swaps ternary branches |
| `RemoveSideEffects` | removes standalone function call statements |

### Operator configuration formats

There are two ways to configure operators in the YAML file.

**`mutators` block** (generated by `init`) — deny-list style. All operators are active by default; set `active: false` on entries you want to skip:

```yaml
mutators:
  - name: RelationalOperatorReplacement
    active: true
  - name: RemoveSideEffects
    active: false   # this operator will be skipped
```

**`operators` list** — allow-list style. Only the listed operators run; everything else is skipped:

```yaml
operators:
  - RelationalOperatorReplacement
  - BooleanLiteralReplacement
```

The two formats are mutually exclusive. If both are present, `operators` takes precedence. The CLI flags `--operator` and `--disable-mutator` always override the file.

## CLI Reference

### Subcommands

```bash
swift-mutation-testing [project-path] init   # generate .swift-mutation-testing.yml
swift-mutation-testing --version             # print version
swift-mutation-testing --help                # print usage summary
```

### Options

```bash
swift-mutation-testing [project-path] [options]
```

| Option | Description |
|---|---|
| `--scheme <scheme>` | Xcode scheme to build and test (required for Xcode, omit for SPM) |
| `--destination <destination>` | `xcodebuild` destination specifier (required for Xcode, omit for SPM) |
| `--target <test-target>` | Limit test execution to this target |
| `--timeout <seconds>` | Per-mutant test timeout (default: 120 Xcode, 30 SPM) |
| `--concurrency <n>` | Parallel workers (default: CPUs − 1) |
| `--no-cache` | Disable result caching |
| `--output <path>` | Write Stryker-compatible JSON report |
| `--html-output <path>` | Write interactive HTML report |
| `--sonar-output <path>` | Write SonarQube generic coverage report |
| `--sources-path <path>` | Root directory for source discovery |
| `--exclude <pattern>` | Exclude files matching glob pattern (repeatable) |
| `--operator <id>` | Activate only this operator (repeatable) |
| `--disable-mutator <id>` | Deactivate this operator (repeatable) |
| `--quiet` | Suppress progress output |

### Common invocations

```bash
# Run with config file (no extra flags needed)
swift-mutation-testing

# Override scheme and destination
swift-mutation-testing --scheme MyApp --destination "platform=macOS"

# Limit to a single test target
swift-mutation-testing --target MyAppTests

# Run only two operators
swift-mutation-testing \
  --operator RelationalOperatorReplacement \
  --operator BooleanLiteralReplacement

# Disable one operator, keep the rest
swift-mutation-testing --disable-mutator RemoveSideEffects

# Exclude generated code
swift-mutation-testing --exclude "Sources/Generated/**"

# Generate all report formats
swift-mutation-testing \
  --output mutation-report.json \
  --html-output mutation-report.html \
  --sonar-output sonar.json

# Suppress progress output (useful in CI)
swift-mutation-testing --quiet

# Invalidate cache and run fresh
swift-mutation-testing --no-cache
```

## Output Formats

### Progress output (default)

Streams build events and per-mutant results to stdout during execution.

```
  ✓ Discovery: 147 mutants (143 schematizable, 4 incompatible) in 2.3s

Building for testing...
  ✓ Built in 18.4s
  ✓ 3 simulators ready

Testing mutants...
  ✓ 1/147  ArithmeticOperatorReplacement  Calculator.swift:12
  ✗ 2/147  RelationalOperatorReplacement  Validator.swift:34
  ⚠ 3/147  RemoveSideEffects              Service.swift:55
  ⏱ 4/147  NegateConditional              Loop.swift:9
```

**Progress icons:**

| Icon | Meaning |
|---|---|
| `✓` | Killed — tests detected the mutation |
| `✗` | Survived — tests passed with the mutation active |
| `⚠` | Unviable — mutant did not compile |
| `⏱` | Timeout — test process exceeded the timeout |
| `–` | No coverage — no test reached the mutated code |

### Text summary (always printed)

```
Results by file:
  Sources/Calculator.swift    score: 95.0%   killed: 19   survived: 1   timeout: 0   unviable: 0
  Sources/Validator.swift     score: 72.4%   killed: 21   survived: 8   timeout: 0   unviable: 0

Survived mutants:
  Sources/Validator.swift:34:5   RelationalOperatorReplacement
  Sources/Validator.swift:67:12  NegateConditional

Overall mutation score: 83.2%
Killed: 122 / Survived: 21 / Timeouts: 0 / Unviable: 4 / NoCoverage: 0
Total duration: 312.7s
```

**Mutation score formula:**

```
score = killed / (killed + survived + timeouts + noCoverage) × 100
```

Unviable mutants are excluded from both numerator and denominator.

### Quiet mode (`--quiet`)

Suppresses all progress output. Only the text summary is printed at the end.

### JSON report (`--output`)

Stryker-compatible JSON report written to the specified path. Includes per-file source content, per-mutant status, and score thresholds.

```bash
swift-mutation-testing --output mutation-report.json
```

### HTML report (`--html-output`)

Interactive per-file dashboard with colour-coded scores and inline survived mutant details.

```bash
swift-mutation-testing --html-output mutation-report.html
```

### SonarQube report (`--sonar-output`)

SonarQube Generic Issue Import Format. Survived mutants are reported as `MAJOR CODE_SMELL` issues; no-coverage mutants as `MINOR`.

```bash
swift-mutation-testing --sonar-output sonar.json
```

### Caching

Results are cached at `<project>/.swift-mutation-testing-cache/results.json`, keyed by SHA256 of the source and test file contents. On subsequent runs, cached mutants skip the build and test cycle entirely.

```
  ✓ Loaded 147 mutants from cache
```

Use `--no-cache` to force a full re-run:

```bash
swift-mutation-testing --no-cache
```

## Suppressing Mutations

Apply the `@SwiftMutationTestingDisabled` attribute to any function, type, or property to exclude all mutations within that scope.

```swift
@SwiftMutationTestingDisabled
func legacyParser(_ input: String) -> Result {
    // All mutations inside this function are suppressed
}

@SwiftMutationTestingDisabled
struct GeneratedModel {
    // All mutations inside this type are suppressed
}
```

**Supported scopes:** functions, initializers, classes, structs, enums, extensions, stored properties.

Suppressed mutations never appear in the report — they are filtered out before execution.

## CI/CD Integration

### GitHub Actions — run on every push

```yaml
- name: Mutation testing
  run: |
    swift-mutation-testing ${{ github.workspace }} \
      --scheme MyApp \
      --destination "platform=macOS" \
      --quiet
```

### GitHub Actions — generate HTML report as artifact

```yaml
- name: Mutation testing
  run: |
    swift-mutation-testing ${{ github.workspace }} \
      --scheme MyApp \
      --destination "platform=macOS" \
      --html-output mutation-report.html \
      --quiet

- name: Upload mutation report
  uses: actions/upload-artifact@v4
  with:
    name: mutation-report
    path: mutation-report.html
```

### GitHub Actions — SonarQube integration

```yaml
- name: Mutation testing
  run: |
    swift-mutation-testing ${{ github.workspace }} \
      --scheme MyApp \
      --destination "platform=macOS" \
      --sonar-output sonar.json \
      --quiet

- name: SonarQube scan
  uses: SonarSource/sonarqube-scan-action@v2
  with:
    args: >
      -Dsonar.externalIssuesReportPaths=sonar.json
```

### GitHub Actions — with config file

```yaml
- name: Mutation testing
  run: swift-mutation-testing ${{ github.workspace }} --quiet
```

When `.swift-mutation-testing.yml` is committed to the repository, no additional flags are needed.

### Speed up CI with caching

Cache the `.swift-mutation-testing-cache` directory between runs to avoid re-testing unchanged mutants.

```yaml
- name: Cache mutation results
  uses: actions/cache@v4
  with:
    path: .swift-mutation-testing-cache
    key: mutation-cache-${{ hashFiles('Sources/**/*.swift', 'Tests/**/*.swift') }}
    restore-keys: mutation-cache-

- name: Mutation testing
  run: swift-mutation-testing ${{ github.workspace }} --quiet
```

### Limit scope in CI

Run only a subset of operators to keep CI fast, and run the full suite locally:

```yaml
- name: Mutation testing (fast)
  run: |
    swift-mutation-testing ${{ github.workspace }} \
      --operator RelationalOperatorReplacement \
      --operator BooleanLiteralReplacement \
      --quiet
```

## Exit Codes

| Code | Meaning |
|---|---|
| `0` | Success |
| `1` | Error — usage error, build failure, or unexpected failure |

Use the exit code in shell scripts:

```bash
swift-mutation-testing --quiet
case $? in
  0) echo "Mutation testing completed" ;;
  1) echo "Error — check scheme, destination, and project path" ; exit 1 ;;
esac
```
